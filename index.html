Board Game

'Board Game', which is a pseudonym, is a work-in-progress digital web-application-based implementation of an existing physical board game, for solo play, or 2-5 person online multiplayer. It is closed source. It consists of:

  • a front-end built using Angular13
  • a backend consisting of a Golang HTTP and Websocket Server which serves the main application, as well as performs game-related api tasks
  • a NodeJs microservice which handles user authentication and authorization
  • a MongoDB database which hosts static game data, as well as user data
  • a discord.js chat bot, which acts as a query interface to the MongoDB instance via message commands

The web application has features such as:

  • a separate-to-the-main-game simple search interface to query cards that exist in the board game by their relevant properties
  • replays, allowing completed games to be viewed again and arbitrarily scrubbed
  • post-game analysis via a few hand-implemented charts (no charting library, just manual geometry calculations and svg)
  • IndexedDB caching of static data (reducing # of network calls on refresh), and some user data, like replays, or current local single player game progress, enabling offline solo play
  • PWA - uses web workers to enable offline usage
  • depends on less npm packages than the default initialized workspace of an Angular project, by manually auditing which we need and removing those deemed unnecessary

Hexyz

Play In Browser

Hexyz is a work-in-progress, open source, tower defense game for Mac, Windows, Linux, and HTML5.
It's written using Lua, and the Amulet game engine. The game is single player, hexagonal-grid, with a few tower types and mob types, and scaling difficulty. The project began as an attempt to develop a library which could generate hexagonal grids, and perform common operations involving them such as:

  • translating mouse coordinates to hex coordinate space and back
  • generating useful Perlin and Simplex noise maps over the grids for procedural terrain generation
  • pathfinding/searching via BFS, Dijkstra, and A*

In testing the library, eventually something resembling a full game was born. It now supports:

  • procedural levels
  • hand-drawn sprites
  • self-composed soundtrack, and sound-fx
  • single game save and load
  • seeded maps and randomness (playing the same map twice has the same enemy spawns, for example)
  • a primitve map editor
  • basic moddability (entities such as mobs and towers are largely stored as simple files which are easy to edit)

Clue

Clue is a work-in-progress, open source, compiled or interpreted general purpose programming language for x64 hardware. It is implemented in C99 with a little bit of C++ on top (mostly for function overloading, and a small number of template classes). It has a minimal dependency philosophy, including as few libraries as possible, even from the C (and certainly C++) standard libraries. It's similar to, and has taken ideas from, the programming languages: C, C++, Java, Lua, Golang, Javascript, Nelua, Crystal, Python, and Jai.
Current key features/properties:

  • fast compile times (~500,000 simple lines in about 4 seconds - compiler isn't complete, so not the best benchmark unfortunately, but it's something)
  • clear, simply-put, and insightful error messages
  • inobtrusive static typing (plenty of type-inference)
  • basic support for metaprogramming
  • a hand written bottom-up shift-reduce parser (no yacc)

This Website and the Webserver Providing It

This page almost got away with having no Javascript, but instead it ended up with just very little, besides those requested by embeds. It's written using a fairly minimal subset of html and css features, in a very ad-hoc way.

The webserver providing these files is a microscopic home-rolled server implemented in pure C using raw Berkeley Sockets (the Linux implementation of them, anyway). It serves a small-medium number of files, and probably isn't secure.

Both the website and the webserver providing it are open source.
